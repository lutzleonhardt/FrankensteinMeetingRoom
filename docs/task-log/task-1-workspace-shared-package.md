### Key Decisions
- **`ExcalidrawElement = unknown` fallback (deviation flagged in plan).**
  The plan explicitly allows this fallback "if importing
  `@excalidraw/excalidraw/element/types` would force adding the
  Excalidraw dep purely to compile shared in M1." That trade-off
  applies here: the field is optional, no consumer touches it until M3,
  and dragging in a heavy Excalidraw dep just for one type is overkill.
  Marked with `// TODO: replace with @excalidraw/excalidraw type when
  M3 lands` so it's hard to miss when M3 starts.
- **`typescript` as a root devDependency, not per-package.** Hoisting
  one TS version keeps Angular shell, esbuild remotes, and shared
  pinned to the same compiler — important once federation gets wired
  up. Pinned `~5.6` (Angular 18/19's supported range; Angular host
  arrives in Task 2 and will lock the actual minor).
- **Added `"lib": ["ES2022", "DOM"]` to `tsconfig.base.json`.** The
  plan didn't list it explicitly, but `bus.ts` references
  `EventTarget`, `CustomEvent`, and `Event` — all DOM types. Without
  `DOM` in `lib`, `tsc --noEmit` errors out. Both shell and remotes
  are browser-only, so DOM in the base is correct, not over-broad.
- **Inline `(globalThis as any).frankensteinBus` cast.** Plan offers
  this or a local interface; I went with the cast for terseness as
  the Key Discoveries note suggests. Avoids a `declare global` block
  that would pollute consumer typings.
- **Seed `weekday(offset, hour)` builds from "this Monday."** Sun→Sat
  handled by `mondayOffset = day === 0 ? -6 : 1 - day`. Times are set
  in local time then serialized to ISO (UTC). Deterministic per-day,
  which the plan explicitly accepts.

— session 2026-05-08
- **Bus payloads typed as `DeepReadonly<T>` instead of cloned at
  runtime.** Codex adversarial review flagged that `on()` hands
  `CustomEvent.detail` to subscribers by reference, so a remote could
  mutate `initialData.attendees` (or nested Excalidraw element arrays)
  and bypass the host's planned `updatedAt`/persistence path.
  Considered three fixes: (a) `structuredClone` at the emit site for
  state-bearing events only; (b) opt-in `clone` flag on `emit`; (c)
  a `DeepReadonly<T>` mapped type on `BusEvents` payloads. Picked (c)
  because the project is strict-TS end-to-end and the boundary is
  intra-window, not intra-process — types are sufficient enforcement
  without a runtime cost on hot paths like `drawing:changed` (Excalidraw
  is debounced upstream of the bus, but cloning per dispatch is still
  pointless cost). Shallow `Readonly<Meeting>` was rejected because it
  leaves nested arrays (`attendees`, `excalidrawData.elements`) mutable.
  Mutable values stay assignable into the readonly emit slot via TS
  covariance, so the host doesn't need casts. Escape hatch if a future
  untyped/JS consumer joins: switch to runtime `structuredClone` then.

### Context for Next Task
Task 2 builds the Angular host in `packages/shell/` with Native
Federation v4 in `dynamic-host` mode. What it can rely on:

- `@frankenstein/shared` resolves via the workspace. Import paths
  are `@frankenstein/shared/bus`, `/types`, and `/seed`. There is no
  `@frankenstein/shared` root entry — only the three subpaths. Any
  bare `from '@frankenstein/shared'` will fail to resolve.
- `bus.ts` exports `emit<K>(name, payload)` and
  `on<K>(name, handler): () => void` (returns an unsubscribe). The
  bus instance is `globalThis.frankensteinBus`, initialized lazily
  via `??=` — re-importing `bus.ts` in another bundle is the by-design
  M2/M3/M4 mechanism for the host to share one bus across remotes.
- Bus payloads are deeply readonly. The host can pass mutable
  `Meeting` objects into `emit('event:selected', { meetingId,
  initialData })` without casts (TS covariance: mutable is assignable
  to readonly). Subscribers receive a deeply-readonly view, so any
  attempted `payload.initialData.attendees.push(...)` is a compile
  error. If the host needs to react to a `drawing:changed` payload by
  storing it on a mutable `Meeting`, copy first (`[...payload.excalidrawData.elements]`
  or `structuredClone`) — don't fight the readonly types with `as any`.
- Federation config in Task 2: `@frankenstein/shared` MUST NOT appear
  in the `shared` block of `federation.config.mjs`. List it as a
  `devDependency` in `packages/shell/package.json` (or filter from
  `shareAll`) so Native Federation's auto-share never picks it up.
  Each bundle compiling its own copy is the design — the bus
  singleton lives in `globalThis`, not in module scope.
- TypeScript is root-hoisted at `~5.6.0`. Angular's CLI may pin a
  narrower version; if Angular's add-schematic upgrades or downgrades
  TS, accept its choice and update the root pin to match.
- `tsconfig.base.json` is the single source of strictness; the shell's
  `tsconfig.json` should `extends` it. The base ships
  `lib: ["ES2022", "DOM"]` already — no need to redeclare.
